# Agents

pwagent ships **10 specialist agents**. Each is a Markdown charter — identity, responsibilities, boundaries, tools, model — that lives in `cli/src/content/agents/<name>/charter.md` and resolves through the [override chain](/architecture/override-chain).

## Why ten, not fifty

The v0.3 simplification cut the roster from 24 down to 10. The criteria for keeping an agent:

1. **Distinct boundary** — its responsibilities don't overlap with another.
2. **Distinct tooling** — it needs a different tool set or model preference.
3. **Common enough** — it's invoked at least once per week per typical user.

Agents that didn't meet all three got merged. For example `triage-ado` and `triage-github` were merged into `triage` because they overlap on `triage` semantics and the model can branch on cwd. `report-weekly` and `report-quarterly` merged into `report` because they only differ in the time window argument.

## How charters compose

Each charter is a single Markdown file with stable section headers. The coordinator parses them with `gray-matter` for frontmatter (`name:`, `description:`) and string-splits for the rest.

```markdown
---
name: triage
description: Classify a Playwright failure into ProductBug / TestCodeBug / Environment / Inconclusive.
---

# Triage

## Identity
- **Role:** Failure triage
- **Project:** pwagent

## Responsibilities
- Read the failure artifact (trace, screenshot, console log)
- Emit a verdict + confidence + brief rationale
- ...

## Boundaries
- You do NOT patch code. That's the heal agent.

## Tools
- read, grep, bash (npx playwright show-trace)

## Model
- Preferred: claude-sonnet-4.5
```

## Browsing them in the CLI

```powershell
pwagent agents list                    # one line per agent
pwagent agents show triage              # full charter
pwagent agents show triage --source     # which override layer won
pwagent agents show triage --raw        # without rendering frontmatter
pwagent agents add ./my-custom.md       # workspace override
```

## Browsing them in the portal

Open `http://127.0.0.1:7337/agents`. The page is server-rendered from `cli/dist/content/agents/*/charter.md`. Each card links to its full charter.

## Reviewer gates

Some agents refuse to spawn without a stamp from a predecessor:

| Agent | Requires |
|---|---|
| `heal` | A stamped `triage` verdict |
| `generate` | (none — can spawn freely) |
| `validate` | (none — runs `playwright test` directly) |
| `plan` | A `triage` or `scenario` artifact |

These are declared in `routing.md` under `## Gates`. The coordinator enforces them at spawn time, refusing the call with a structured error.
