# a11y

The **a11y** agent is the full-stack accessibility specialist. It scans URLs and repos for WCAG violations, reviews code for contrast, color, links, rendering modes, responsive viewports, and interactive elements, auto-fixes violations, verifies fixes against ADO bugs, generates Playwright accessibility tests, and produces self-contained HTML compliance reports.

It applies **Microsoft Accessibility Standards (MAS)** on top of WCAG 2.1 AA — stricter large-text contrast (4.5:1), focus indicator contrast (3:1), touch targets (44×44 px minimum), and Windows Narrator/UIA compatibility.

## Identity

- **Role:** Accessibility specialist (scan · review · fix · verify · test-gen · report)
- **Project:** pwagent
- **Default model:** Opus 4.7 or Sonnet 4.6 (accessibility audits benefit from deep reasoning — do not downgrade)

```
› @pwagent-a11y --review contrast --path src/components
› @pwagent-a11y --review color --path src/
› @pwagent-a11y --review links --path src/
› @pwagent-a11y --review modes --path src/
› @pwagent-a11y --review viewports --path src/
› @pwagent-a11y --review interactive --url http://localhost:3000
› @pwagent-a11y --fix src/components/LoginForm.tsx
› @pwagent-a11y --verify-fix http://localhost:3000/login color-contrast AB#54321
› @pwagent-a11y --test-gen --component src/components/Modal.tsx
› @pwagent-a11y --report --url http://localhost:3000
```


## Modes

| Flag | What it does |
|---|---|
| `--scan <url>` | Run axe-cli against the URL; output grouped by impact (critical → minor) |
| `--scan-repo --base-url <url>` | Discover routes, batch-scan all pages, consolidated report |
| `--review contrast --path <dir>` | Extract colors, verify 4.5:1 / 3:1 ratios, suggest compliant alternatives |
| `--review color --path <dir>` | Detect WCAG 1.4.1 — information conveyed by color alone |
| `--review links --path <dir>` | Detect WCAG 2.4.4 — generic/ambiguous link text |
| `--review modes --path <dir>` | Check 8 MAS modes (HC Black/White, Forced Colors, Reduced Motion, Zoom, Text Spacing, Dark) |
| `--review viewports --path <dir>` | Check 7 viewports (320px–2560px) — overflow, touch targets, iOS zoom triggers |
| `--review interactive --url <url>` | Generate and run Playwright tests for 9 interactive element types |
| `--fix <file-or-dir>` | Auto-apply ARIA fixes with confidence scoring (High / Medium / Low) |
| `--verify-fix <url> <rule> <bug>` | Re-scan, compare against baseline, post result to ADO bug |
| `--test-gen --component <file>` | Write `tests/accessibility/<component>.spec.ts` with axe-core + keyboard + ARIA tests |
| `--report --url <url>` | Generate self-contained `accessibility-report.html` |

## Standards applied

MAS is loaded before every analysis session and overrides WCAG where stricter:

| Area | WCAG | MAS |
|---|---|---|
| Contrast — normal text | 4.5:1 | 4.5:1 |
| Contrast — large text | 3:1 | **4.5:1** |
| Contrast — UI components | 3:1 | 3:1 |
| Focus indicator | Any visible change | 3:1 against adjacent color |
| Touch targets | Not specified | **44×44 px minimum** |
| Screen reader | WCAG 4.1.2 | Windows Narrator + NVDA tested |

## Critical rules

<Callout type="warning">
**Never assume the ADO org, repo URL, or test environment URL.** The a11y agent operates across multiple repos and environments. Any of the three that are not explicitly provided must be asked for — a wrong assumption creates an artifact in the wrong place.
</Callout>

1. **ADO org** — always ask before posting bug comments or updating work item state.
2. **Repo URL / local path** — always ask before creating branches or committing.
3. **Test environment URL** — always ask before running axe or Playwright; do not assume localhost is running.
4. Never auto-create ADO bugs or open PRs without explicit user approval.
5. Never skip post-fix verification — `--verify-fix` before marking any bug resolved.

## Boundaries

- Does not own the validate pipeline — `pwagent-validate --a11y` handles pre/post-fix axe deltas wired into the CI chain.
- Does not triage S360 WCAG action items — `pwagent-s360` owns those.
- Does not merge PRs — `pwagent-publish` does that.
- Does not modify CI pipelines.
