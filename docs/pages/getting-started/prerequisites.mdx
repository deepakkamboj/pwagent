# Prerequisites

pwagent is a coordinator â€” most heavy lifting happens through CLIs it shells out to and MCP servers it calls as tools. **Both CLIs and MCP servers are required before first use.** The `pwagent prereqs` command detects, reports, and (with `--install`) installs the missing CLI tools.

<Callout type="warning">
**Mandatory onboarding step.** Run `pwagent prereqs` before your first agent invocation. Missing prereqs cause silent failures â€” agents simply can't spawn the tools they need.
</Callout>

### Checking and installing

```powershell
pwagent prereqs                       # report only â€” no network changes
pwagent prereqs --install             # interactive â€” prompts before each install
pwagent prereqs --install --yes       # non-interactive, accept all
pwagent prereqs --install --include-optional --yes
pwagent prereqs --install --skip "kusto"
```

## MCP servers

MCP servers are Claude Code tool providers â€” not CLIs. Agents call them via `mcp__<server-id>__<tool-name>` patterns. They are configured in Claude Code's MCP settings (`~/.claude/claude_desktop_config.json` or the workspace-level `.claude/settings.json`), not via `pwagent prereqs --install`.

<Callout type="error">
**Required for S360 and ADO-linked accessibility work.** `pwagent-s360`, `pwagent-a11y --verify-fix`, and `pwagent-publish` are non-functional without at least one ADO MCP configured. `pwagent-s360 --fix` additionally requires Kusto MCP and S360 MCP.
</Callout>

| Server ID | Label | Required by | What it enables |
|---|---|---|---|
| `kusto` | Kusto MCP | `pwagent-s360`, `pwagent-analyze`, `pwagent-triage` | KQL queries â€” S360 action items, flake history, pipeline telemetry |
| `s360` | S360 MCP | `pwagent-s360` | Update S360 action item ETA and status after a fix is verified |
| `dynamicscrm-repo` | ADO MCP â€” DynamicsCRM | `pwagent-s360`, `pwagent-a11y`, `pwagent-publish` | Create/update ADO bugs + PRs in `dynamicscrm.visualstudio.com` |
| `msazure-repo` | ADO MCP â€” msazure | `pwagent-s360`, `pwagent-a11y`, `pwagent-publish` | Create/update ADO bugs + PRs in `msazure.visualstudio.com` |

Configure the ADO MCP for the org your workspace repo lives in â€” you don't need both.

### Setup: Kusto MCP

Add to `~/.claude/claude_desktop_config.json` under `mcpServers`:

```json
{
  "mcpServers": {
    "kusto": {
      "command": "npx",
      "args": ["-y", "@microsoft/kusto-mcp"],
      "env": {
        "KUSTO_CLUSTER": "https://s360prodro.kusto.windows.net",
        "KUSTO_DATABASE": "service360db"
      }
    }
  }
}
```

The S360 agent uses `s360prodro.kusto.windows.net / service360db`. The analyze agent uses the PowerApps-Engineering cluster â€” override `KUSTO_CLUSTER` per invocation or add a second server entry.

Full setup guide: [aka.ms/kusto-mcp-setup](https://aka.ms/kusto-mcp-setup)

### Setup: ADO MCP

Install the one that matches your org. Replace `<your-personal-access-token>` with a PAT that has `Work Items (Read & Write)` and `Pull Requests (Read & Write)` scopes.

**DynamicsCRM org** (`dynamicscrm.visualstudio.com`):

```json
{
  "mcpServers": {
    "dynamicscrm-repo": {
      "command": "npx",
      "args": ["-y", "@microsoft/ado-mcp"],
      "env": {
        "ADO_ORG_URL": "https://dynamicscrm.visualstudio.com",
        "ADO_PAT": "<your-personal-access-token>"
      }
    }
  }
}
```

**msazure org** (`msazure.visualstudio.com`):

```json
{
  "mcpServers": {
    "msazure-repo": {
      "command": "npx",
      "args": ["-y", "@microsoft/ado-mcp"],
      "env": {
        "ADO_ORG_URL": "https://msazure.visualstudio.com",
        "ADO_PAT": "<your-personal-access-token>"
      }
    }
  }
}
```

Full setup guide: [aka.ms/ado-mcp-setup](https://aka.ms/ado-mcp-setup)

### Verify MCPs are live

After adding MCP entries, restart Claude Code and run:

```
pwagent doctor --mcps
```

`pwagent doctor` probes each MCP server by calling a lightweight discovery tool and reports âś“ / âś— per server. A âś— here means the server didn't start or the config key is wrong â€” check `~/.claude/mcp.log` for the startup error.

## Checking everything at once

```powershell
pwagent doctor
```

`pwagent doctor` runs both the CLI prereq matrix and the MCP server probes in one pass. You should see `Ready.` and a full features matrix. Fix any âś— before running agents.

## Per-workspace config

After prereqs are green, each workspace needs a minimal config file:

```powershell
copy pwagent.config.example.json .pwagent\config.json
```

For S360 work, also add `.pwagent/s360.config.md`:

```markdown

adoProject: OneCRM
```

Edit [`cli/src/prereqs/matrix.ts`](https://github.com/microsoft/pwagent/blob/main/cli/src/prereqs/matrix.ts) â€” append a `Prereq` object with `id`, `label`, `reason`, `tier`, `detect`, `installers`, `unlocks`. For MCP servers edit [`cli/src/prereqs/mcps.ts`](https://github.com/microsoft/pwagent/blob/main/cli/src/prereqs/mcps.ts) â€” append an `MCPServer` object.

The vitest suite in [`cli/tests/prereqs.test.ts`](https://github.com/microsoft/pwagent/blob/main/cli/tests/prereqs.test.ts) catches incomplete shapes on both files.

## Common issues

| Symptom | Cause | Fix |
|---|---|---|
| `gh auth (logged in)` reports âś— after `gh auth login` | Token missing `read:user` scope | `gh auth refresh -s read:user` |
| `kusto CLI` reports âś— but binary is on PATH | Detector uses `kusto.cli` not `kusto` | Symlink `kusto.cli` to PATH as `kusto` |
| `@axe-core/cli` reports âś— after global install | npm prefix not on PATH | `npm config get prefix` â†’ add `<prefix>/bin` to PATH |
| `pwagent doctor --mcps` shows âś— for `kusto` | Server didn't start | Check `~/.claude/mcp.log`; verify `KUSTO_CLUSTER` env var |
| ADO MCP âś— | PAT expired or missing scopes | Regenerate PAT with `Work Items (R/W)` and `Pull Requests (R/W)` |
| `npx playwright --version` non-zero exit | Deprecation shim returns 1 | pwagent trusts the version regex over the exit code â€” auto-recovered |
