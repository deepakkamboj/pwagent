# Prerequisites

pwagent is a coordinator â€” most heavy lifting happens through other CLIs it shells out to. The `pwagent prereqs` command detects, reports, and (with `--install`) installs the missing ones.

## Reporting only

```powershell
pwagent prereqs                       # report; no network changes
```

Reads `which` / `--version` for each entry. Returns exit 0 if everything required is present, 1 otherwise.

## Installing missing ones

```powershell
pwagent prereqs --install             # interactive â€” prompts before each install
pwagent prereqs --install --yes       # non-interactive, accept all
pwagent prereqs --install --include-optional --yes
pwagent prereqs --install --skip "azure-cli,kusto-cli"
```

## Platform detection

| Platform | Detected by | Falls back to |
|---|---|---|
| Windows | `winget --version` â†’ `winget install` | manual link |
| macOS | `brew --version` â†’ `brew install` | manual link |
| Debian/Ubuntu | `apt -v` + sudo available â†’ `sudo apt install` | manual link |
| Fedora/RHEL | `dnf --version` + sudo available â†’ `sudo dnf install` | manual link |
| Arch | `pacman --version` + sudo available â†’ `sudo pacman -S` | manual link |
| Node globals | always `npm install -g` | n/a |
| gh extensions | always `gh extension install` | n/a |

## Safety rules

- **No silent installs.** Default is report-only. `--install` requires the flag.
- **No sudo without explicit consent.** Linux installs show the exact `sudo` line and wait for confirmation.
- **Network-only when needed.** Report mode makes no network calls â€” only `which` / `--version` probes.

## Common issues

| Symptom | Cause | Fix |
|---|---|---|
| `gh auth (logged in)` reports âś— even after `gh auth login` | Token doesn't include `read:user` scope | `gh auth refresh -s read:user` |
| `kusto CLI` reports âś— but `kusto.cli` is on PATH | Detector uses `kusto` not `kusto.cli` | Symlink or add `kusto.cli` to PATH as `kusto` |
| `@axe-core/cli` reports âś— after `npm i -g @axe-core/cli` | npm prefix differs from PATH | `npm config get prefix` and add `<prefix>` to PATH |
| `npx playwright --version` non-zero exit | Deprecation shim returns 1 | pwagent trusts the version regex over the exit code â€” auto-recovered |
