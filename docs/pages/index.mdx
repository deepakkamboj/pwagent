# pwagent

**Multi-agent Playwright testing — Squad design, GitHub Copilot SDK runtime.**

**`pwagent`** = **`pw`** + **`agent`** — short for **"playwright agent"**, mirroring the `pw` two-letter prefix used by Playwright's own packages.

pwagent is a standalone CLI that ports every design pattern from [Brady Gaster's Squad](https://github.com/bradygaster/squad) — charters, routing, reviewer gates, ceremonies, parallel-by-default — but ships its **own embedded runtime**. One binary, one mental model, one update path. No Copilot CLI host process, no per-session coordinator-manifest token tax.

It runs entirely on **GitHub Copilot via [`@github/copilot-sdk`](https://www.npmjs.com/package/@github/copilot-sdk)** — Microsoft-internal Copilot license powers the model calls; no external API keys.

## Works with everything you already use

pwagent is a **plain Node CLI** — anything that can run a binary can run pwagent. No IDE lock-in, no extension to install.

| Surface | Status | How |
|---|---|---|
| Standalone terminal (PowerShell, bash, zsh) | ✓ first-class | `pwagent <cmd>` |
| Claude Code CLI | ✓ shell-out | Run from inside a Claude Code session — Claude's bash tool invokes it |
| Claude Code VS Code extension | ✓ shell-out | Extension's integrated terminal runs `pwagent` directly |
| GitHub Copilot CLI | ✓ shell-out | Copilot CLI's shell can invoke `pwagent` (and vice-versa) — they coexist |
| GitHub Copilot VS Code extension | ✓ shell-out | Extension's terminal runs `pwagent` like any binary |
| VS Code, Cursor, Windsurf, JetBrains | ✓ tasks + terminal | `tasks.json` entries or integrated terminal |
| CI runners (GitHub Actions, ADO Pipelines) | ✓ first-class | Node 22 + `GH_TOKEN` configured |

**Key principle:** pwagent is **not** an IDE plugin or VS Code extension. It's a binary. We keep it that way to avoid lock-in. See [FAQ → Where does pwagent run?](/faq#where-does-pwagent-run-which-ide--tool-integrations) for details.

## Quick Start

```powershell
# Install and link the CLI
cd D:\gith\pwagent
npm install
npm run build
npm link --workspace cli

# Verify prerequisites and authenticate
pwagent prereqs --install
pwagent login
pwagent doctor

# Run an agent
pwagent run triage --run-id 12345

# Launch the portal
pwagent portal start            # http://127.0.0.1:7337

# Launch the docs (this site!)
npm run dev:docs                # http://127.0.0.1:7338
```
